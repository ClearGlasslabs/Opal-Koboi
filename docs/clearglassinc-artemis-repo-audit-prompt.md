# ClearGlassInc Artemis Repository Audit Prompt

Use this prompt when you want a principal-level codebase review that stays focused on high-leverage, production-safe platform upgrades.

## Final polished version

Act as a principal software architect, AI systems designer, and product strategist.

Audit my repository and upgrade it into a future-tech platform for **ClearGlassInc Artemis** and founder **Desmond Otieno Odhiambo**. Make the codebase more powerful, secure, scalable, intelligent, and production-ready without unnecessary bloat.

## Main objective

**Treat the repo like a defense-grade system that must scale under pressure and evolve continuously.**

Turn this repo from a working project into a serious technical platform with:

- clean architecture
- advanced automation
- AI capabilities
- strong security
- observability
- scalability
- maintainability

## What to review

Find:

- missing advanced features
- weak or placeholder code
- duplicate or unnecessary code
- architectural bottlenecks
- automation opportunities
- AI workflow opportunities
- security gaps
- observability gaps
- performance bottlenecks
- poor abstractions
- modules that should be simplified, split, or removed

## What to build toward

Add only features that create real leverage, such as:

- AI agents and orchestration
- context-aware workflows
- retrieval / knowledge layers
- telemetry, logging, and monitoring
- event-driven automation
- secure auth and access control
- secrets handling
- retries and fallback logic
- benchmarking and evaluation
- plugin-style modular architecture
- CI/CD and deployment automation
- caching and performance optimization
- test coverage and code quality enforcement
- documentation for scale and handoff

## Working rules

- Prefer high-impact improvements over cosmetic changes
- Keep the system simple where possible
- Do not break existing functionality
- Do not add features just to look advanced
- Remove anything unnecessary or slow
- Make the repo easier to maintain, extend, and trust
- Treat this like a production platform, not a demo

## Output format

Return your work in this structure:

### 1. Repository assessment

What the repo does well, what is missing, and what is blocking it from becoming top-tier.

### 2. Best upgrade opportunities

List and rank the highest-value features to add.

### 3. Refactor plan

Show what files, modules, or systems need work, what to remove, what to simplify, and what to build next.

### 4. Implementation plan

For each major upgrade, explain:

- purpose
- architecture
- dependencies
- risks
- testing approach
- rollout sequence

### 5. Future direction

Explain how to make the repo feel more advanced, more intelligent, more secure, and more aligned with ClearGlassInc Artemis and Desmond Otieno Odhiambo.

## Short version

Audit my repo for ClearGlassInc Artemis and Desmond Otieno Odhiambo. Remove weak code, find the most important upgrades, and improve it into a secure, modular, observable, AI-enabled, production-ready platform.
